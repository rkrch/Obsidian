### Этап 1: Укрощение Linux и терминала

`ssh ubuntu@172.20.0.177` gfhjkm56
Mine gfhjkm65

[[Minecraft server]]

1.  **Приручите терминал.** 90% администрирования делается через него.
    *   Базовые команды: [[ls]], [[cd]], [[pwd]], [[cp]], [[mv]], [[rm]], [[mkdir]], [[cat]], [[nano]], [[vim]].
    *   Работа с правами: [[chmod]], [[chown]], [[sudo]].
    *   Поймите разницу между пользователем и [[root]].
    *   Работа с процессами: [[ps]], [[top]], [[htop]], [[kill]].
    *   Поиск файлов: [[find]], [[grep]].
	- `Ctrl + C` - прервать выполнение команды
	- `Ctrl + D` - *выйти из терминала*


2.  **Управление пакетами.** Это основа поддержания системы в рабочем состоянии.
    *   Освойте apt update && apt upgrade.
    *   Установите несколько программ через [[apt]] (например, `htop`, `wget`, `curl`).
    *   Попробуйте установить программу не из репозиториев, вручную скачав `.deb` пакет.

3.  **Работа с сетевыми утилитами.**
    *   `ip addr` (или устаревший `ifconfig`) — просмотр сетевых интерфейсов.
    *   `ping` — проверка доступности узла.
    *   `ss` (или `netstat`) — просмотр открытых портов и соединений.
    *   `traceroute` — трассировка маршрута.

4.  **Настройка виртуальной сети в VirtualBox.**
    *   Поэкспериментируйте с разными режимами сетевых адаптеров в настройках виртуальной машины (VM):
        *   **NAT:** По умолчанию. VM "сидит" за NAT вашей хост-системы. Извне до нее не достучаться.
        *   **Сетевой мост (Bridge):** VM ведет себя как отдельный компьютер в вашей локальной сети. У нее будет свой IP-адрес. **Идеально для экспериментов с сетевыми службами.**
        *   **Внутренняя сеть (Internal Network):** Несколько VM могут общаться друг с другом, но не с внешним миром.
