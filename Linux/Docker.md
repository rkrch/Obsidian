### Погружение в Docker



1.  **Установка Docker.**
    *   Не используйте `sudo apt install docker.io` (там часто старая версия). Следуйте официальной инструкции для Ubuntu: установите `docker-ce` из репозитория Docker.
    *   Добавьте своего пользователя в группу `docker`, чтобы не писать `sudo` перед каждой командой (`sudo usermod -aG docker $USER` и перелогиньтесь).

2.  **Освойте базовые команды.**
    *   `docker pull` — скачать образ.
    *   `docker images` — список образов.
    *   `docker run` — запустить контейнер. Поэкспериментируйте с ключами `-it` (интерактивный режим), `-d` (в фоне), `--name`, `-p` (проброс портов), `-v` (монтирование томов).
    *   `docker ps` — список работающих контейнеров.
    *   `docker exec -it <container_id> /bin/bash` — зайти в запущенный контейнер.
    *   `docker stop`, `docker rm`, `docker rmi` — управление жизненным циклом.

3.  **Практические задачи с Docker:**
    *   **Запустите веб-сервер nginx:** `docker run -d -p 8080:80 --name my-webserver nginx`. Откройте в браузере на хосте `http://localhost:8080`. Поздравляю, вы только что подняли свой первый сервис!
    *   **Запустите простенькое веб-приложение:** Найдите на Docker Hub образ какого-нибудь готового приложения (например, `wordpress`). Запустите его, свяжите с контейнером базы данных (`mysql`). Вам придется использовать `--link` или, что лучше, `docker network`.
    *   **Напишите свой простой `Dockerfile`:** Создайте файл, который соберет образ с вашим простым Python/Node.js скриптом.

### Этап 3: Администрирование сетей и сервисов (здесь будет самый ценный опыт)

Теперь объединим знания Linux и Docker для построения небольшой "лабораторной" сети.

1.  **Настройте статический IP-адрес.** В режиме "Сетевой мост" зайдите в `/etc/netplan/`, отредактируйте конфиг и примените его (`sudo netplan apply`). Теперь у вашей VM будет постоянный IP в вашей домашней сети.

2.  **Поднимите свой SSH-сервер.**
    *   Установите `openssh-server`: `sudo apt install openssh-server`.
    *   Подключитесь к своей же VM с хостовой машины по SSH: `ssh user@<ip_address_of_vm>`. Это фундаментальный навык удаленного администрирования.

3.  **Разверните LAMP/LEMP стек вручную (без Docker!).** Это классика жанра.
    *   **LAMP:** Установите и настройте Apache (`apache2`), PHP (`php`), MySQL (`mysql-server`).
    *   **LEMP:** Вместо Apache поставьте Nginx (`nginx`).
    *   Создайте простой PHP-сайт, подключитесь к базе данных. Научитесь настраивать виртуальные хосты (в Nginx — `sites-available`/`sites-enabled`).

4.  **Создайте свою мини-сеть с помощью Docker Compose.**
    *   Установите `docker-compose`.
    *   Напишите `docker-compose.yml` файл, который поднимает:
        *   Веб-приложение (например, Wordpress)
        *   Базу данных (MySQL)
        *   Обратный прокси (например, Nginx), который перенаправляет запросы к веб-приложению.
    *   Запустите всё одной командой `docker-compose up -d`. Это даст вам понимание оркестрации сервисов.

5.  **Поэкспериментируйте с сетевыми политиками и фаерволом.**
    *   Изучите `ufw` (Uncomplicated Firewall) — это простой фронтенд для `iptables`.
    *   Включите его: `sudo ufw enable`.
    *   Настройте правила: разрешите только SSH (порт 22) и HTTP (порт 80), а все остальные порты закройте. Проверьте, что ваш веб-сервер доступен, а другие порты (например, 8080) — нет.

### Идеи для более сложных проектов:

*   **Настройте VPN-сервер** (например, WireGuard) внутри VM и подключитесь к своей "домашней" сети извне.
*   **Разверните свой Git-сервер** (Gitea или GitLab) с помощью Docker.
*   **Постройте свой мониторинг:** Свяжите связку Prometheus + Grafana в Docker, чтобы мониторить ресурсы вашей VM.
*   **Настройте резервное копирование** важных данных с VM на хост-машину или в облако с помощью `rsync` или `borgbackup`.
*   **Поднимите свой DNS-сервер** (например, Pi-hole в Docker) для блокировки рекламы на уровне всей сети.

### Важные советы на старте:

1.  **Ломайте систему!** Это виртуальная машина, ее всегда можно откатить к снапшоту. Не бойтесь экспериментировать и случайно "сломать" что-то. Сделайте **снапшот (snapshot)** в VirtualBox прямо сейчас, в чистом состоянии, и перед каждым крупным экспериментом.
2.  **Гуглите ошибки.** Умение читать логи (`journalctl`, `tail -f /var/log/syslog`) и искать по ним решения — ключевой навык.
3.  **Документируйте.** Заведите себе файлик `notes.md` и записывайте туда удачные команды, конфиги и решения проблем. Это ваша личная база знаний.
4.  **Используйте официальную документацию.** Для Docker, Ubuntu, Nginx — всегда сначала смотрите на официальные сайты, там самая точная информация.

Удачи в изучении! Этот путь даст вам огромное количество практических навыков, востребованных в IT.
