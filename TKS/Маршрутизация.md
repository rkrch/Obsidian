## Маршрутизация 2 
Маршрутизация - это поиск маршрута доставки пакета сетями через транзитные узлы - маршрутизаторы.

Этапы маршрутизации:
- изучение сети (какие подсети есть, какие маршрутизаторы, как они объединены)
- продвижение пакетов на маршрутизаторе (когда сеть изучена и на маршрутизатор поступил пакет, и для него надо определить, куда отправить)

Если мы передаем пакет на маршрутизатор, то нужно знать, какой именно из маршрутизаторов, подключенных к этой сети, выбрать для передачи пакета. Эту информацию маршрутизатор хранит в таблице маршрутизации.

## Динамическая маршрутизация 
— это процесс автоматического обмена маршрутной информацией между маршрутизаторами с использованием специальных протоколов маршрутизации. Она упрощает администрирование сетей, особенно больших и сложных, и обеспечивает автоматическое обновление маршрутов при изменении топологии.

Классификация протоколов маршрутизации

Протоколы динамической маршрутизации можно классифицировать по нескольким признакам:

1. По области действия:
	1. Внутренние протоколы маршрутизации (IGP — Interior Gateway Protocols):  
	    Используются для обмена маршрутами внутри одной автономной системы (AS).  
		- RIP (Routing Information Protocol)    
		- OSPF (Open Shortest Path First)    
		- EIGRP (Enhanced Interior Gateway Routing Protocol)    
		- IS-IS (Intermediate System to Intermediate System)    
	
	2. Внешние протоколы маршрутизации (EGP — Exterior Gateway Protocols):  
	    Применяются для обмена маршрутами между разными автономными системами.  
		- BGP (Border Gateway Protocol)    

2. По алгоритму работы:
	1. Протоколы на основе дистанционно-векторного алгоритма (Distance-Vector):		    
		- Обмениваются информацией о расстоянии до сети и направлении (векторе).	    
		- Периодически отправляют таблицы маршрутизации своим соседям.		    
		- Просты в настройке, но медленно сходятся.  
		    - RIP		    
			- IGRP		    
		
	2. Протоколы на основе алгоритма состояния канала (Link-State): 
		- Каждое устройство строит карту всей сети (топологию) и рассчитывает маршруты на её основе.	    
		- Быстро сходятся, но требуют больше ресурсов.      
			- OSPF
			- IS-IS
		
	3. Гибридные протоколы (Hybrid):
		- Сочетают элементы Distance-Vector и Link-State.  
			- EIGRP
	    
3. По способу представления адресов:
	1. Классовые протоколы (Classful):
		- Не поддерживают VLSM (Variable Length Subnet Mask) и CIDR (Classless Inter-Domain Routing).
		- Передают маршруты без информации о маске подсети.  	
			- RIP v1
			- IGRP
		
	2. Бесклассовые протоколы (Classless):
		- Поддерживают VLSM и CIDR.
		- Передают информацию о маске подсети вместе с маршрутом.  	
			- RIP v2	    
			- OSPF	    
			- EIGRP	    
			- BGP	    
	
4. По способу обновления информации:
	1. Протоколы с периодической отправкой таблиц (Periodic Updates):
		- Периодически отправляют полные таблицы маршрутизации соседним устройствам.  
			- RIP
		
	2. Протоколы с триггерным обновлением (Triggered Updates):
		- Отправляют изменения в маршрутах только при изменении топологии.  
			- OSPF
			- EIGRP

**Краткое описание популярных протоколов**

- **RIP (Routing Information Protocol):**
    Простой протокол с использованием метрики "количество хопов". Медленная сходимость и ограничение в 15 хопов.    
- **OSPF (Open Shortest Path First):**
    Протокол состояния канала, использует алгоритм SPF (Shortest Path First). Подходит для больших сетей.    
- **EIGRP (Enhanced Interior Gateway Routing Protocol):**  
    Гибридный протокол от Cisco, сочетает простоту Distance-Vector с функциями Link-State. Быстро сходится.    
- **IS-IS (Intermediate System to Intermediate System):**  
    Протокол состояния канала, схожий с OSPF, но чаще используется в провайдерских сетях.    
- **BGP (Border Gateway Protocol):**  
	Основной протокол маршрутизации в Интернете. Позволяет обмениваться маршрутами между автономными системами.
